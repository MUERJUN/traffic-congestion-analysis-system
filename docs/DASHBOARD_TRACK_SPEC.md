# Track B：Streamlit系统展示规格

## 1. 分支与定位

- 计划分支：`feature/streamlit-dashboard`
- 产品定位：基于 METR-LA 历史数据的交通分析与风险预测回放平台。
- 禁止表述：实时交通系统、实时监控、实时预警或已接入城市交通管理系统。
- 本轮状态：仅形成规格，不实现页面或业务代码。

## 2. 用户与展示目标

- 交通管理/分析人员：快速识别历史高风险时段和道路观测点，理解模型为什么形成判断。
- 教师/评审：验证数据挖掘闭环、时间序列方法、模型对比和解释可信度。
- 演示者：通过固定历史案例完成稳定、连贯、可复核的展示。

## 3. 信息架构

系统包含四个页面：

1. 项目介绍
2. 数据分析
3. 预测展示
4. 分析报告

全局必须持续显示数据时间范围和“历史数据回放，非实时系统”声明。导航、颜色、风险阈值和术语在四页保持一致。

## 4. 页面规格

### 4.1 项目介绍页面

展示：

- 项目目标、使用场景和完整分析闭环。
- METR-LA 数据范围、207个传感器、五分钟粒度。
- 当前能力与限制：历史回放、单一数据源、无实时接入、无因果事故诊断。
- 页面使用指引和风险等级说明。

### 4.2 数据分析页面

展示：

- 数据概览：时间范围、传感器数、有效观测、0值/缺测情况。
- 拥堵或低速趋势：全局与选定传感器趋势。
- 时间规律：小时曲线、星期×小时热力图、工作日/周末对比。
- 道路风险排行：拥堵率、持续时间或低速指标，并显示有效覆盖率。
- 拥堵前兆：事件前60分钟趋势与对照。
- 图表下方的定量业务解读和限制。

筛选项建议包括日期范围、工作日/周末、传感器和排行数量。筛选后结论必须同步更新，不能保留与图表不一致的静态文字。

### 4.3 预测展示页面

输入：

- 必填：道路传感器编号，使用可搜索下拉框。
- 历史回放时点：由系统提供有效历史时点或代表案例，禁止默认成“当前时间”。

输出：

- 所选历史时点、当前有效速度和当前状态。
- 未来30分钟拥堵概率。
- 低/中/高风险等级。
- 最近60分钟速度趋势。
- 3—5个主要影响因素、实际特征值、推动方向和通俗解释。
- 数据质量/置信度提示及对应业务建议。

当历史窗口不足、关键数据缺失、模型或解释产物不存在时，页面必须进入明确空状态，不得自动填充伪造概率。

### 4.4 分析报告页面

展示：

- 数据、拥堵标签和时间切分说明。
- Logistic Regression、Random Forest、XGBoost 五项指标对比。
- 混淆矩阵、ROC曲线及最终模型选择理由。
- 全局特征重要性和因素方向。
- 高峰/非高峰、工作日/周末及传感器分组误差分析。
- 基于证据的业务建议、数据限制和模型限制。

## 5. 风险等级与视觉规范

- 低风险：`[0, 0.40)`，绿色。
- 中风险：`[0.40, 0.70)`，橙色。
- 高风险：`[0.70, 1.00]`，红色。
- 如 Track A 经验证集确认调整阈值，Dashboard只能读取正式配置，不自行硬编码另一套口径。
- 颜色之外同时使用文字/图标，避免只依靠颜色传递状态。

## 6. 数据与模型对接

Track B 只消费 Track A 的版本化产物，不在页面内重复训练、重新估计自由流速度或重算测试指标。

最低对接字段：

- EDA：时间、传感器、聚合指标、有效样本量、缺测率。
- 预测：传感器、历史时点、当前速度、概率、风险等级、模型版本。
- 解释：因素名、特征值、贡献方向、通俗描述。
- 报告：模型名、Accuracy、Precision、Recall、F1、ROC-AUC、最终选择标记。

读取产物时应验证模式版本和必要字段；不兼容时显示错误说明，不静默使用旧结果。

## 7. 性能与缓存

- 页面读取预计算汇总与已训练模型，不在交互请求中执行全量EDA或训练。
- 大型数据按页面需要裁剪或聚合，使用 Streamlit 缓存但不得缓存错误版本。
- 普通电脑上的主要交互目标为3秒内完成；首次模型加载可单独记录。

## 8. 异常与空状态

- 无效传感器：提示并提供合法选择，不抛出原始异常。
- 所选时点缺测：说明无法预测或自动引导到最近有效历史时点，不能隐式填补。
- 模型文件缺失/版本不符：停止预测并说明所需产物。
- 图表无数据：显示筛选条件和无数据原因。
- 解释不可用：保留概率展示时必须明确“解释产物不可用”，不能生成通用套话冒充解释。

## 9. 业务建议展示规则

- 建议必须映射到正式分析发现或模型解释。
- 使用“建议关注、建议评估、需人工复核”等辅助决策语言。
- 不输出未经数据验证的信号配时、限速值或收益百分比。
- 不把时间、速度或历史同期特征描述为事故、天气等具体因果事件。

## 10. 测试重点

- 四页面导航与非实时声明。
- 代表传感器、边界时间、缺测时间和非法输入。
- 风险阈值边界0.40与0.70。
- 页面指标与 Track A 报告逐项一致。
- 相同输入重复预测结果稳定。
- 无模型产物时不展示虚构结果。
- 图表标题、单位、图例、筛选与结论一致。

具体验收条件见 `docs/ACCEPTANCE_CRITERIA.md`。
