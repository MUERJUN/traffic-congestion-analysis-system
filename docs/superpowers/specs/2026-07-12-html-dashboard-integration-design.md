# HTML 正式版仪表盘集成设计

## 1. 目标

将现有 Streamlit 仪表盘作为功能与数据口径基线，建设独立 HTML 正式版前端。正式版必须保留原有筛选、图表交互、表格查看、地图回放和预测解释能力，同时统一视觉层级，避免形成静态展示页或第二套不一致的数据逻辑。

## 2. 产品边界

- 系统定位仍为 METR-LA 历史交通数据回放与分析平台，不描述为实时交通系统。
- Python 继续负责数据读取、清洗结果加载、指标计算和前端数据构建。
- HTML、CSS 与 JavaScript 负责正式页面布局和交互展示。
- Streamlit 页面保留为功能对照和开发验证入口，不作为正式前端入口。
- 不引入天气数据，不改变现有模型与拥堵标签口径。

## 3. 页面结构

正式版保留四个业务页面：

1. 项目介绍：项目定位、数据基线、分析闭环和使用边界。
2. 数据分析：日期与时间筛选、拥堵趋势、时间规律、道路风险排行、速度趋势和道路走廊热力地图。
3. 预测展示：传感器与历史时点选择、拥堵概率、风险等级、速度变化和主要影响因素。
4. 分析报告：模型指标对比、特征重要性、业务结论和交通优化建议。

页面采用统一顶栏与侧边导航，桌面端保持信息密度，窄屏自动折叠导航和卡片布局。

## 4. 图表交互

图表统一使用 Plotly 渲染，并开启适用的交互工具：

- 滚轮或工具栏缩放；
- 拖动和平移；
- 框选与套索选择；
- 悬停显示中文字段与单位；
- 重置坐标轴和自动缩放；
- 全屏或展开查看；
- 下载 PNG 图片。

所有标题、坐标轴、图例、悬停内容和注释使用可理解的中文业务名称，不直接暴露内部字段名。

## 5. 表格交互

Python 生成的表格数据以 JSON 或 JavaScript 数据对象形式提供给 HTML。前端表格至少支持：

- 点击列标题排序；
- 关键词搜索；
- 分页与每页数量选择；
- 横向滚动；
- 全屏或展开查看；
- 风险等级、概率与指标的中文格式化；
- 空数据状态和数据口径说明。

首批迁移四张核心表格：道路风险排行、预测主要影响因素、模型指标对比、特征重要性排行。

## 6. 地图展示

地图以道路走廊线段表达风险，不使用大面积圆点遮挡底图。道路颜色对应拥堵风险，悬停显示道路或传感器标识、历史时点、速度和拥堵概率。地图支持缩放、拖动、重置视野与图层说明。

当数据只包含传感器坐标、缺少精确道路几何时，页面明确标注为“基于相邻传感器连线的道路走廊近似展示”，不得将近似线段宣称为真实道路拓扑。

## 7. 数据流

数据流固定为：

`METR-LA 与模型产物 -> Python 数据构建脚本 -> data.js/JSON -> HTML 组件 -> 用户筛选与交互`

前端不得重新定义拥堵标签、风险等级阈值或模型评价口径。数据构建失败时保留上一次有效产物，并显示可定位的构建错误；页面加载不到数据时显示中文错误状态，不静默展示空图。

## 8. 视觉规范

- 使用交通管理类仪表盘风格，主色为深蓝，风险色仅用于状态和重点数据。
- 页面背景、卡片、边框和文字建立清晰层级，避免通篇深色大字号。
- 数字指标突出，说明文字弱化；同类卡片保持统一高度与间距。
- 表格、图表和地图共享相同的标题区、工具区和口径说明样式。
- 不使用无业务含义的装饰动画、渐变堆叠或大面积高饱和色块。

## 9. 验收标准

- 四个页面均可从 HTML 正式入口访问。
- 原 Streamlit 中的核心筛选与分析功能在 HTML 中有对应实现。
- Plotly 图表具备缩放、悬停、重置和下载能力。
- 四张核心表格具备排序、搜索、分页、横向滚动和展开能力。
- 所有用户可见业务字段均为中文，内部英文变量不直接展示。
- 地图使用道路走廊着色，并明确数据几何的真实性边界。
- 数据构建脚本、JavaScript 语法和项目自动化测试全部通过。
- 在 1366px 及以上桌面宽度下无内容遮挡；窄屏下导航和内容可正常阅读。

## 10. 不在本轮范围

- 实时数据接入与实时预警；
- 用户账号、权限和数据库后台；
- 天气、事故或其他外部数据融合；
- 修改模型训练流程与重新训练模型；
- 将近似道路走廊升级为经过地图匹配验证的真实道路拓扑。
