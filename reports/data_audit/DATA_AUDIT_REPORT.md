# METR-LA 数据审计报告

## 1. 审计结论

核心文件可正常读取，结构与 METR-LA 基准特征一致：34,272个连续五分钟时点、207个传感器，共7,094,304个速度单元。时间轴、传感器编号、位置元数据和邻接矩阵之间不存在结构性错位，可以进入 EDA 准备阶段。

当前最大的质量风险不是时间戳缺失，而是数据将大量疑似缺测编码为数值0。0值占全部速度单元的8.1094%，且有2,148个时点全部207个传感器同时为0。这类全网同步0在业务上更符合采集缺失或预处理占位，而非所有道路瞬时完全停车。因此 EDA 不得将所有0直接计入真实低速/拥堵统计。

| 正速度最小值 | 0.3333 mph |
| 正速度中位数 | 63.2222 mph |
| 正速度均值 | 58.4597 mph |
| 正速度第1/99百分位 | 13.125 / 69.75 mph |

最大值恰好为70且存在较多70值，提示数据可能存在上界截断或速度量化；现阶段只登记，不做删除。大幅跳变中相当一部分预计由0值进入或退出造成，EDA前应区分“与0相邻的跳变”和“两个正值之间的跳变”。

## 7. 元数据一致性

- HDF5 的207个 ID 与 `graph_sensor_ids.txt` 内容和顺序完全一致。
- 位置表含207个唯一 ID，无核心传感器缺位置，也无多余位置记录。
- 邻接矩阵为207×207，ID和顺序完全一致，含1,722个非零权重。
- 距离表无缺失或负成本，但包含4,106个传感器，是构造目标子图的上游网络表，EDA地图只使用与207个核心 ID 相关的记录。

## 8. 进入EDA前的数据决策

1. 原始 HDF5 永不修改。
2. 将0增加为“疑似缺测”标记，不立即视为拥堵。
3. EDA主口径对速度统计排除0，同时单独展示0值/缺测覆盖；敏感性版本再展示“保留0”会造成多大偏差。
4. 全网同步0区间直接列为系统性缺测候选，不参与道路拥堵排行。
5. 单传感器长连续0区间列为点位缺测候选。
6. 时间戳暂按无时区本地墙上时间保存，并在文档中声明；未经来源证据不擅自进行UTC转换。
7. 在确认清洗规则前不计算拥堵率、不生成标签、不训练模型。
