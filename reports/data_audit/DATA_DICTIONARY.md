# METR-LA 数据字典（审计版）

## 1. 核心 HDF5

文件：`data/raw/metr-la.h5`

HDF5 保存的是 Pandas DataFrame 固定格式，内部结构如下：

| HDF5 节点 | 形状 | 类型 | 含义 |
|---|---:|---|---|
| `/df/axis0` | 207 | bytes，最长6字符 | DataFrame 列名，即传感器编号 |
| `/df/axis1` | 34,272 | int64 | DataFrame 时间索引，以纳秒时间戳保存 |
| `/df/block0_items` | 207 | bytes | 数值块对应的传感器列名 |
| `/df/block0_values` | 34,272 × 207 | float64 | 速度观测矩阵；行是时间，列是传感器 |

### 逻辑字段

| 字段 | 类型 | 单位/时区 | 可空性 | 审计备注 |
|---|---|---|---|---|
| `timestamp` | datetime | 文件未携带时区；数据文献通常按洛杉矶当地时间理解，当前暂不强制赋时区 | 无时间缺口 | 2012-03-01 00:00 至 2012-06-27 23:55，严格五分钟递增 |
| `sensor_id` | string | 无 | 不可空 | 207个唯一编号；与同源 ID 文件及邻接矩阵顺序一致 |
| `speed` | float64 | mph（按数据集文献口径） | 无 IEEE NaN，但存在大量0 | 正值范围 0.3333—70；0需按缺测候选处理，不能直接解释为真实停车速度 |

## 2. 传感器位置

文件：`data/metadata/graph_sensor_locations.csv`

| 字段 | 类型 | 含义 |
|---|---|---|
| `index` | integer | 0—206 的内部顺序 |
| `sensor_id` | string | 传感器编号 |
| `latitude` | float | 纬度 |
| `longitude` | float | 经度 |

共有207行、207个唯一传感器，和 HDF5 传感器集合完全一致。

## 3. 道路距离

文件：`data/metadata/distances_la_2012.csv`

| 字段 | 类型 | 含义 |
|---|---|---|
| `from` | string | 起点传感器编号 |
| `to` | string | 终点传感器编号 |
| `cost` | float | 作者数据中的道路网络距离/代价 |

文件共有295,374条记录、4,106个不同起点和终点；它是构造207节点子图的上游距离表，不应误称为只有207个传感器之间的完整笛卡尔表。无负代价或缺失代价。

## 4. 邻接矩阵

文件：`data/metadata/adj_mx.pkl`

加载后为三元素列表：传感器 ID 列表、ID 到索引映射、207×207 邻接权重矩阵。矩阵非零元素1,722个，数值范围0—1，ID 顺序与 HDF5 完全一致。
