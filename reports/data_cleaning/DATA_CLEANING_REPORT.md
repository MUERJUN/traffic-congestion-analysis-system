# Phase 3A 数据清洗报告

## 1. 范围与结论

本阶段建立了可复现、非破坏性的 METR-LA 清洗视图。原始 HDF5 保持只读，未进行原位修改；清洗结果保留 `speed_raw`，并将大于0的有限速度写入 `speed_valid`，原始0值转为缺测候选。未进行插值、未来窗口标签、训练集切分或模型训练。

清洗后的长表保持7,094,304行，与34,272个时间点×207个传感器严格一致，可以进入正式EDA和后续特征工程。

每次运行在读取数据前检查原始文件哈希；哈希不一致时停止，不对未知版本继续处理。

## 3. 清洗规则

### 3.1 原始值与有效值

- `speed_raw`：完整保存原始值。
- `speed_valid`：仅保留有限且大于0的速度；0转为缺测候选。
- 不删除行，不把0直接改写为拥堵状态。
- 不执行前向、后向、线性或历史均值填补。

### 3.2 质量标记

| 标记 | 规则 |
|---|---|
| `is_zero_observation` | 原始速度等于0 |
| `is_systemwide_zero` | 同时刻207个传感器全部为0 |
| `is_majority_zero` | 同时刻至少50%传感器为0 |
| `is_long_sensor_zero_run` | 单传感器连续0至少12步，即60分钟 |
| `is_large_jump` | 相邻5分钟速度绝对变化超过40 mph |
| `is_large_jump_from_zero` | 大跳变的当前值或前值为0 |
| `has_valid_location` | 传感器经纬度完整 |

60分钟长零值阈值用于识别质量风险，不是拥堵阈值，也不会把短于60分钟的0自动认定为真实停车。

## 4. 清洗统计

| 指标 | 结果 |
|---|---:|
| 正速度有效观测 | 6,519,002（91.8906%） |
| 0值/缺测候选 | 575,302（8.1094%） |
| 全网同步0时点 | 2,148 |
| 至少半数传感器为0时点 | 2,149 |
| 长连续0中的观测 | 521,515 |
| 长连续0占全部0值 | 90.6506% |
| 超过40 mph的大跳变 | 47,405 |
| 与0相邻的大跳变 | 46,100（占97.2471%） |
| 两个正值之间的大跳变 | 1,305 |

大部分0值处于至少60分钟的连续零值段，且97.25%的大跳变与0值进入/退出相邻。这进一步支持“0主要是缺测/占位候选”的处理原则。如果直接把0当作真实速度，会把数据恢复时的跳变误判为交通状态骤变。

## 6. 防泄漏说明

- 本阶段只构造当前时点的质量标记和日历字段。
- 未读取预测时点之后的速度来填补当前数据。
- 描述性EDA自由流速度只用于全样本统计，不会复用为模型特征。
- 后续建模时必须重新只用训练期数据估计自由流速度、历史同期统计和任何预处理参数。
